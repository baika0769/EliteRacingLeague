# AGENTS.md

## Quy tắc bắt buộc

* Luôn trả lời bằng tiếng Việt.
* Ưu tiên trả lời ngắn, đúng việc, không lan man.
* Mặc định chỉ đọc và phân tích.
* Không được sửa file, tạo file mới, xóa file, chạy build/test/install nếu người dùng chưa nói rõ: "Tôi đồng ý cho sửa".
* Không in toàn bộ code hoặc toàn bộ file ra terminal nếu không được yêu cầu.
* Khi cần sửa file, chỉ hỏi ngắn: "Bạn có đồng ý cho tôi sửa file không?"
* Trước khi sửa, chỉ trình bày kế hoạch tối đa 8 dòng:

  1. File sẽ sửa
  2. Nội dung sửa chính
  3. Cách kiểm tra
* Chỉ được sửa đúng các file đã nêu trong kế hoạch.
* Không sửa file khác ngoài kế hoạch.
* Không tự ý đổi tên class, hàm, namespace, file hoặc folder nếu không bắt buộc.
* Không tự ý sửa file .env, appsettings.json, connection string, API key, password hoặc secret.
* Không chạy lệnh nguy hiểm như xóa file, reset git, clean database hoặc drop database.
* Nếu cần sửa thêm ngoài kế hoạch, phải dừng lại và hỏi người dùng trước.

## Quy tắc tiết kiệm token

* Chỉ đọc file liên quan trực tiếp đến nhiệm vụ.
* Không phân tích toàn bộ dự án nếu không được yêu cầu.
* Không tạo bảng dài nếu không cần.
* Không giải thích lý thuyết dài.
* Sau khi làm xong, chỉ trả lời:

  1. File đã tạo/sửa
  2. Lệnh kiểm tra nên chạy
  3. Kết quả hoặc lỗi còn lại
  4. Bước tiếp theo
